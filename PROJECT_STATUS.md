# 🚀 프로젝트 진행 상황 및 가이드 (PROJECT_STATUS)

이 문서는 `pdf_masker` 프로그램 랜딩 페이지 웹사이트의 구축 내역과 향후 유지보수를 위한 가이드를 상세히 기록한 문서입니다. 나중에 사이트를 수정하실 때 이 문서를 참고하시면 매우 편리합니다.

## 1. 프로젝트 개요
- **목적**: 100% 오프라인 문서 개인정보 마스킹 프로그램의 소개 및 다운로드를 제공하는 랜딩 페이지
- **기술 스택**: React (TypeScript) + Vite + Tailwind CSS
- **디자인 컨셉**: 직관적이고 시원시원한 초대형 텍스트 및 레이아웃, 심플하고 신뢰감 있는 디자인

## 3. 🛠 나중에 쉽게 수정하는 방법 (유지보수 가이드)

### 3.1. 텍스트 내용이나 링크를 바꿀 때
프로그램 이름, 설명, 다운로드 링크 등 주요 데이터는 한 곳에 모아두었습니다.
- **수정 위치**: `src/constants.ts` 파일
- 해당 파일을 열어서 한글 문구, 버전 이름(`v1.0.0`), 파일 이름 등을 글자만 바꾸시면 사이트 전체에 즉시 반영됩니다.
- 본문 중간의 큼직한 인사말 등을 바꾸고 싶다면 `src/App.tsx` 파일을 열어 수정하세요.

### 3.2. 다운로드 프로그램을 새 버전으로 교체할 때
프로그램 성능이 개선되어 새 파일로 갈아 끼워야 할 경우:
1. 기존 `public/downloads/` 폴더 안에 있는 예전 `.exe` 및 `.dmg` 파일을 삭제합니다.
2. 새로 만든 프로그램 파일을 같은 이름으로 폴더 안에 넣습니다.
3. (만약 파일 이름이 변경되었다면, `src/constants.ts` 안의 경로 이름도 똑같이 고쳐주세요.)

### 3.3. 수정한 내용을 실제 인터넷 웹사이트에 적용(배포)할 때
로컬에서 수정을 완료했다면, **VS Code 하단의 터미널** 창에 아래 명령어 3줄을 한 줄씩 치면 알아서 업데이트됩니다.
```bash
# 1. 변경된 모든 파일을 장바구니에 담기
git add .

# 2. 장바구니에 메모장 붙여서 포장하기 (메모 내용은 자유롭게 적기)
git commit -m "프로그램 새 버전으로 교체함"

# 3. GitHub으로 쏘아 올리기
git push -u origin main
```
- 위 3줄을 입력하시고 **약 1~2분 정도 기다리시면** 깃허브 로봇이 알아서 사이트를 새로고침하여 전 세계 인터넷에 적용해 줍니다!
